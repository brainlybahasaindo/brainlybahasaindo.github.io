Cara Menggunakan Mysqldump Triggers Hanya Untuk Tahun 2023

Apa itu mysqldump?

mysqldump adalah alat yang dapat digunakan untuk membuat cadangan salinan dari basis data MySQL. Ini berguna jika Anda ingin memiliki cadangan yang tersedia untuk memulihkan di masa mendatang jika terjadi kerusakan atau setidaknya menjaga data Anda tetap aman. Mysqldump mengekstrak data dan struktur dari basis data dan menyimpannya dalam file SQL yang dapat Anda gunakan untuk memulihkan basis data.

Mysqldump juga dapat digunakan untuk membuat salinan atau cadangan dari seluruh basis data atau tabel tertentu. Anda dapat menggunakan kueri yang khusus untuk membuat cadangan hanya dari tabel tertentu. Itu memudahkan Anda untuk mencadangkan tabel yang berbeda secara terpisah dari basis data.

Cara Menggunakan Mysqldump Triggers

Mysqldump triggers adalah fitur yang memungkinkan Anda untuk membuat cadangan dari tabel yang berubah. Ini berguna ketika Anda ingin membuat cadangan hanya dari tabel yang berubah. Ini juga berguna jika Anda ingin mengekstrak hanya data yang berubah dari tabel tertentu.

Untuk menggunakan mysqldump triggers, Anda harus membuat trigger terlebih dahulu. Trigger adalah perintah yang dijalankan setiap kali terjadi suatu peristiwa. Tanpa trigger, Anda harus mengekstrak semua data dari basis data secara manual. Trigger akan membuatnya lebih mudah untuk mengambil hanya data yang berubah dari basis data.

Mengatur Trigger

Untuk mengatur trigger, Anda harus terlebih dahulu membuat sebuah file SQL yang berisi kode berikut:

DELIMITER //
 CREATE TRIGGER backup_trigger
 AFTER UPDATE ON tabel
 FOR EACH ROW
 BEGIN
 SET @filename = CONCAT('/path/to/backup/', tabel, '_', DATE_FORMAT(NOW(), '%Y-%m-%d_%H-%i-%s'), '.sql');
 SET @command = CONCAT('mysqldump tabel -u root -p password > ', @filename);
 PREPARE stmt FROM @command;
 EXECUTE stmt;
 DEALLOCATE PREPARE stmt;
 END //
 DELIMITER ;

Kode di atas akan secara otomatis membuat cadangan dari tabel yang terlibat dalam peristiwa yang ditentukan. Anda dapat mengubahnya sesuka hati sesuai kebutuhan Anda. Anda juga dapat menambahkan lebih banyak trigger untuk membuat cadangan dari berbagai tabel.

Menjalankan Mysqldump

Setelah Anda selesai membuat trigger, Anda dapat menjalankan mysqldump. Untuk menjalankannya, buka terminal dan jalankan perintah berikut:

$ mysqldump -u root -p password database > /path/to/backup/database.sql

Perintah di atas akan mengekstrak semua data dari basis data dan menyimpannya dalam file SQL. Anda dapat menggunakan file ini untuk memulihkan basis data.

Menggunakan mysqldump Triggers

Setelah Anda selesai membuat trigger, Anda dapat menjalankan mysqldump. Untuk menjalankannya, buka terminal dan jalankan perintah berikut:

$ mysqldump -u root -p password --triggers database > /path/to/backup/database.sql

Perintah di atas akan mengekstrak semua data yang disimpan dalam trigger. Ini memungkinkan Anda untuk mengekstrak hanya data yang berubah dari basis data.

Kesimpulan

Mysqldump triggers adalah alat yang berguna bagi mereka yang ingin mengekstrak hanya data yang berubah dari basis data. Ini membuatnya lebih mudah untuk mencadangkan basis data Anda. Anda hanya perlu mengatur trigger dan menjalankan mysqldump dengan opsi yang benar untuk membuat cadangan.
